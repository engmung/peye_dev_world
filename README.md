# 🔥 피아이개발팀 — 깃허브 놀이터

🌐 **[레포 탐색기 바로가기 →](https://engmung.github.io/peye_dev_world/)**

> 코딩 에이전트와 함께 깃/깃허브를 몸으로 배우는 일주일 실험

## 👋 우리는

| 이름 | 상태 |
|------|------|
| 승훈 | 에이전트로 풀스택 해봄. 협업은 처음 |
| 채빈 | 에이전트로 게임 만들어봄. 깃은 처음 |
| 예인 | 에이전트로 게임 만들어봄. 깃은 처음 |

## 📌 이건 뭐하는 곳?

목적 없이 자유롭게 놀면서 깃과 깃허브를 체득하는 공간이야.
폴더 만들고, 글 쓰고, 코드 올리고, 남의 거 건드려보고, 부서지면 고치고.
**실수하는 게 목적**이야. 여기서 뭘 망가뜨려도 아무 문제 없어.

## 🗓️ 진행 방식

### Phase 1 (Day 1~4): 자유놀이 — main 직접 push

- 각자 자기 이름으로 폴더 만들기 (`승훈/`, `채빈/`, `예인/`)
- 거기에 아무거나 올리기 — 메모, 코드, 낙서, 뭐든
- 남의 폴더 구경하기, 건드려보기도 OK
- **충돌나면 좋은 거야.** 그게 배우는 거

### Phase 2 (Day 5~7): 브랜치 + PR 체험

- `git checkout -b 이름/작업내용` 으로 브랜치 만들어서 작업
- GitHub에서 Pull Request 올려보기
- 서로 PR에 코멘트 달아보기
- merge 해보기

### 마지막 날: 회고

- `git log --oneline --graph --all` 로 일주일 궤적 보기
- Gource로 시각화 영상 만들기
- 각자 뭘 배웠는지 공유

## 🛠️ Git 기본 명령어

이것만 알면 돼. 진짜야.

### 매일 쓰는 것들

```bash
# 깃허브에서 레포 처음 가져오기 (맨 처음 한 번만)
git clone https://github.com/engmung/peye_dev_world.git

# 지금 내 상태 확인 (수시로 쳐봐)
git status

# 파일을 스테이지에 올리기 (= "이번에 이거 기록할게")
git add 파일이름        # 특정 파일만
git add .              # 전부 다 (주의: 잡파일도 올라감)

# 스냅샷 찍기 (= 세이브 포인트)
git commit -m "여기에 뭐 했는지 적기"

# 깃허브에 올리기
git push

# 다른 사람이 올린 거 가져오기 (push 전에 항상!)
git pull
```

### 브랜치 (Phase 2에서)

```bash
# 새 브랜치 만들고 이동
git checkout -b 이름/작업내용

# 브랜치에서 작업 후 push
git push origin 이름/작업내용

# → 그다음 GitHub 웹에서 PR 만들기
```

### 문제 생겼을 때

```bash
# push가 안 될 때 → pull 먼저
git pull

# 충돌 났을 때 → 파일 열어서 <<<<<<< ======= >>>>>>> 찾아서 정리 → add → commit

# 뭘 잘못했는지 모르겠을 때
git status    # 현재 상태 확인
git log       # 지금까지 기록 확인

# 에이전트한테 물어봐도 돼! 근데 명령어는 직접 쳐보자
```

## ⚠️ 딱 하나 절대 하지 말 것

```bash
git push --force   # ← 이거 절대 금지. 다른 사람 기록 날아감
```
